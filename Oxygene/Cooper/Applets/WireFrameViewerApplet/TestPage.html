Oxygene for Java Wire Frame Viewer applet test page

This page shows the Oxygene for Java Wire Frame Viewer applet running in the browser, assuming you have the Java Runtime Environment installed.

Note that the applet tag used to display the applet uses the codebase attribute to access the compiled .jar file in the bin\Debug folder. This means it expects you to run it from the Debug configuration. If you wish to test the release configuration you will need to update the codebase attribute to point to the bin\Release folder instead.

When deploying your applet, if it resides in the same folder as the web page that refers to it then the codebase attribute is not required.

Note also that Java applets have a tendency to be cached. This means that after first displaying your applet, any further changes you make to the code and then compile to a .jar will likely not be shown when you re-display the applet (for example in this test page). To control the cache you can use the Java console. To show the Java Console choose the Java option in Control Panel, switch to the Advanced tab, expand the Java Console node in the tree of options and select 'Show console'. Whenever a Java Platform applet is running the Java Console will now appear and you can press X on the keyboard to clear the classloader cache.

Click and drag on the applet to rotate the cube.

( You need to enable Java to see this applet. )
